- `MEMORY_INDEX_GATE_PROBE=1`

- `MEMORY_PROCESS_STATS=1`
- `MEMORY_INDEX_RECONCILE_DEBOUNCE_MS=0`
- `MEMORY_INDEX_RECONCILE_INTERVAL_MS=0`

Assertions recorded in each JSON artifact:

- `/api/index-status` observed `building` before reconcile and `ready` with `chunkCount > 0` after cold indexing.
- `/api/search` returns at least one result with `source: "index"`.
- Search latency while reconcile runs: p50 <= 50 ms, p95 <= 200 ms, p99 <= 500 ms, max <= 1000 ms.
- Reconcile-owner and service process memory include `rss`, `external`, `arrayBuffers`, `heapUsed`, and V8 `usedHeapSize`.
- Reconcile-owner and service RSS stay <= 1.5 GiB.
- V8 `usedHeapSize` stays below the Task 0 corpus-proportional guard.
- Event-loop delay and DB/WAL/SHM byte sizes are recorded.
- Cold full-index wall time is checked against the 10 minute target and 20 minute hard cap.
- Legacy `loadSearchCorpus` sentinel has zero invocations in index mode.
