Post-Mortem: Q4 Migration Outage

The recent migration attempt resulted in significant downtime exceeding our SLA.

Finding: Attempting to migrate all legacy data synchronously caused massive transaction locks. Future migrations of this scale must utilize an asynchronous, dual-write strategy.

We learned that prioritizing strict consistency over availability in this context was a critical error.
