Evaluating Voyage AI for Dense Retrieval

calendar_today Created: Oct 12, 2023
update Updated: 2d ago
verified Confidence: High
#embeddings #llm-ops

After testing multiple embedding providers (OpenAI, Cohere, Pinecone's proprietary), we are standardizing on Voyage AI for our dense retrieval pipelines moving forward.

Context

Our memory system requires highly accurate semantic search over technical documentation, code snippets, and unstructured personal notes. Traditional BM25 lexical search falls short on conceptual queries, while existing OpenAI text-embedding-ada-002 models struggle with highly domain-specific technical jargon without fine-tuning.

Evaluation Criteria

  • Retrieval Accuracy (NDCG@10): Must outperform baseline models on our internal eval dataset.
  • Latency: p95 latency under 150ms for embedding generation.
  • Context Window: Needs to handle chunk sizes up to 4096 tokens efficiently.

Implementation Detail

We will be integrating the voyage-2 model. The client setup is straightforward via their Python SDK:

import voyageai

vo = voyageai.Client(api_key="VOYAGE_API_KEY")

# Embed a single technical query
query = "How to configure Tailwind with Vite?"
query_embedding = vo.embed(
    [query], 
    model="voyage-2", 
    input_type="query"
).embeddings[0]

print(f"Embedding dimension: {len(query_embedding)}") # 1024

Note the use of input_type="query". For indexing documents, we must ensure we use input_type="document" to maintain optimal vector space separation.
