## Mission

The `sync.uncommitted-vault` verify warn keeps reoccurring no matter how often the operator drains the vault. Today alone they drained it twice (one for 14 files, one for 22). Root cause is structural, not transient:

1. **`autoCommitRawsIfDirty` (`src/sync/auto-commit-raws.ts`) only commits `raw/`. If ANY non-raw file is dirty, the worker bails with `skipped-non-raw-dirty`.** No partial progress.
2. Normal system operations constantly write non-raw paths: `memory compile` writes `state/compile-state.json` + `state/scheduled-compile-prompt.md` + `log.md`; auto-link writes wiki frontmatter; `sync-prompts --apply` writes `prompts/*.md`; audit logs append to `wiki/.audit/llm-<date>.md`; auto-heal appends to `embeddings/auto-heal.jsonl`.
3. The worker bails → backlog accumulates silently → operator drains by hand → cycle restarts on the next non-raw write.

Fix the loop, not the symptom. Two complementary changes:

- **A. Extend the auto-push worker to commit a curated whitelist of non-raw system-managed paths** alongside `raw/`, after the same secret-shape scan.
- **B. Move ephemeral state out of git entirely** — `state/scheduled-compile-prompt.md` is a transient digest (and has triggered secret-shape redaction in the past); `state/compile-state.json` is a runtime cache. Neither belongs in commit history. Move both to a gitignored runtime path; update every reader.

Both kept conservative. The worker must still **fail-soft on a true non-whitelist dirty file** so the operator can still be alerted to anything unexpected.

## Phase 3 — Options + trade-offs

**A — Worker whitelist (commit on behalf of operator)**

- A1. **Whitelist a fixed set of system-managed paths** (`log.md`, `wiki/.audit/`, `embeddings/auto-heal.jsonl`, auto-link wiki writes) and commit them with `raw/` in a single commit per worker pass. Apply `containsSecretShape` to each whitelisted file before staging; on a hit, skip-and-warn the same as raw secret-shape behavior. *Recommended.*
- A2. **Two separate commits per worker pass** (one `chore: auto-capture N raw`, one `chore: auto-state N files`). Cleaner history; slightly more commits.
- A3. **Stricter: only auto-commit if EVERY dirty file is in the whitelist OR raw/.** Anything else (genuine wiki edits, config changes) still triggers `skipped-non-raw-dirty` and operator must commit. Keeps the alert function intact. **Combine with A1.**

**B — Move ephemeral state out of git**

- B1. **Move `state/scheduled-compile-prompt.md` and `state/compile-state.json` to `var/compile/` (gitignored).** Update every reader (CLI, dashboard, tests, scheduler). `git rm` from history; add `var/` to `.gitignore`.
- B2. **Same but rename `var/` to `.cache/` or `runtime/`.** Bikeshed; pick one.
- B3. **Keep state/ tracked; just add to the worker whitelist.** Simpler but `state/scheduled-compile-prompt.md` keeps re-triggering secret-shape risk; history bloats.

Recommend **A1 + A3 + B1**. Operator alert is preserved (anything outside whitelist still warns), ephemeral state stops bloating history and stops triggering secret-shape, system-managed files commit cleanly without operator drain.
