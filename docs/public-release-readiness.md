### Seed corrections and misses

- The seed was right that markdown-on-disk is canonical and embeddings are JSONL sidecars; the code backs that with path-derived markdown and JSONL stores, not a DB. Evidence: src/storage/paths.ts:16-99; src/retrieval/embeddings-store.ts:31-80.
- The seed missed that the current CLI search command is dashboard-backed and can be offline even when local files exist. Evidence: src/cli/commands/search.ts:40-63; src/cli/commands/search.ts:148-156.
- The seed missed the LLM provider abstraction: OpenRouter and Ollama are configurable separately from the embedder. Evidence: src/llm/factory.ts:8-90.
- The seed missed the VS Code install target. Evidence: src/cli.ts:110-126; src/cli/commands/install/vscode.ts:53-75.
- The seed understated the package-surface risk: `npm pack --dry-run --json` currently includes 4,257 files, including ignored `.claude/worktrees`, tests, docs, source, and dist, because there is no `files` whitelist or `.npmignore`; npm emitted `gitignore-fallback No .npmignore file found`. Evidence: package.json:1-44 plus pack command output from this audit.
- Several command families are explicitly registered as stubs, so the public CLI surface is broader than the implemented behavior. Evidence: src/cli.ts:1137-1174.

## 4. Installability gap analysis

### What first-run does today

- `memory init` uses `MEMORY_ROOT` or `~/.memory`, creates the root and subdirectories, renders `templates/schema.md`, writes prompts/index/log/config/preferences, and initializes a git repo when possible. Evidence: src/storage/paths.ts:16-99; src/cli/commands/init.ts:131-290.
- Template values are detected from global git config and source repo HEAD; there is no interactive wizard, no readline flow, and no user choice of tools/provider/storage in the init path. Evidence: src/cli/template-render.ts:28-49; src/cli/commands/init.ts:156-164.
- `memory install <platform>` writes hook/MCP config for one target tool; Codex install writes config TOML hooks plus an MCP server block, Claude Code install writes settings/plugin wiring, Antigravity install writes MCP/config/plugin files, and VS Code uses user profile/extension paths. Evidence: src/cli/commands/install/codex.ts:30-74; src/cli/commands/install/claude-code.ts:44-90; src/cli/commands/install/antigravity.ts:30-57; src/cli/commands/install/vscode.ts:53-75.

### Keyless first run

- `memory init` itself does not require Voyage, OpenAI, or OpenRouter keys. Evidence: src/cli/commands/init.ts:131-290.
- The default template config sets `embedder.provider: voyage` and `model: voyage-4-large`, so embedding refresh/provider tests will require `VOYAGE_API_KEY` unless the user changes config. Evidence: templates/config.yaml:12-14; src/retrieval/embedder/factory.ts:26-58.
- Search retrieval catches embedder refresh/query failures and degrades to lexical/exact/graph/metadata paths, so local dashboard search can still produce non-vector results when an embedder key is missing. Evidence: src/retrieval/search.ts:228-268; src/retrieval/search.ts:271-353.
- CLI `memory search` is not keyless-local by itself because it only calls the dashboard API; if no dashboard is running it exits 3 and suggests `memory grep`. Evidence: src/cli/commands/search.ts:40-63; src/cli/commands/search.ts:148-156.
- LLM features are optional unless `llm` config exists; OpenRouter needs `OPENROUTER_API_KEY`, Ollama can use a host/default. Evidence: src/llm/factory.ts:41-90.
